Overview

The gxUINT16 class is used to represent 16 bit-unsigned integers independently of the operating system or hardware platform used. It works by separating a 16-bit value into two separate byte values and reordering the bytes lowest-order to highest-order. An gxUINT16 type has a base 10 positive limit of 65,535.

Conditional Directives

__USE_NATIVE_INT_TYPES__ - This conditional directive allows the use of native integer types in place of platform interoperable integer types for debugging purposes only.

Functions

The gxUINT16 class is designed to function in the same manner as built-in integer types and includes a full complement of arithmetic and comparison operator overloads. Each operator is overloaded for the following data types: gxUINT16, __LWORD__, __ULWORD__, __WORD__, __SWORD__, __ULWORD__, __USWORD__, __SBYTE__, and __UBYTE__.
